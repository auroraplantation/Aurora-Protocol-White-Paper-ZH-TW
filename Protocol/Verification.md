# 驗證架構

Aurora Protocol 採用**五層驗證架構**，以彌合鏈上智能合約與鏈下農業操作之間的信任差距。每一層處理不同維度的風險，並在批次生命週期的不同階段發揮作用。

---

## 信任問題

純數位資產的 DeFi 協議可完全在鏈上驗證狀態。實體資產協議面臨本質不同的挑戰：底層資產存在於物理世界，其狀態無法被智能合約原生讀取。

Aurora Protocol 必須回答區塊鏈本身無法回答的問題：

- 這座農場是否真實存在？
- 這位發起人的身份是否屬實？
- 作物是否確實已播種？
- 收穫是否確實在進行？
- 發起人是否確實會履行還款？

五層驗證架構旨在針對上述各維度提供遞進式的信心保障。

---

## 五層架構

### 第一層 — 發起人身份驗證

**時機：** 批次建立前
**內容：** 驗證發起人的法律身份、企業註冊及營運歷史

本層確認發起人為真實、可識別且具備可驗證歷史紀錄的實體。Aurora Labs 執行以下工作：

- 依據當地登記機構核實企業註冊資訊
- 核心負責人的身份驗證
- 營運歷史評估（經營年限、過往融資紀錄）
- 銀行帳戶及支付管道驗證
- 供應鏈合作夥伴的徵信查詢（如適用）

**產出：** 發起人通過或未通過批次建立資格審核。

### 第二層 — 資產與營運驗證

**時機：** 批次結構化期間，部署前
**內容：** 驗證底層農業操作及其經濟可行性

本層確認融資申請由真實且經濟可行的農業操作支撐。評估包括：

- 實地驗證（現場考察或衛星影像確認）
- 作物類型及生長季節驗證
- 該地區及作物類型的歷史產量資料
- 供應鏈銷售安排（買方合約或市場通路確認）
- 財務模型審閱（成本結構、預期收入、利潤率分析）

**產出：** 批次參數定案並核准部署。

### 第三層 — 里程碑證據驗證

**時機：** 批次生命週期中每個里程碑階段
**內容：** 即時驗證里程碑條件是否達成，作為資金釋放的前提

本層是營運層面最關鍵的環節。每個里程碑階段，發起人提交證明里程碑已達成的證據。證據類型包括：

- **里程碑 1（播種）：** 播種確認照片、種子及農資採購收據、農藝師報告
- **里程碑 2（生長）：** 生長階段檢查照片、衛星影像更新、農資施用紀錄
- **里程碑 3（收穫）：** 收穫文件、倉儲收據、買方交貨確認、銷售發票

Aurora Labs 透過以下方式組合驗證證據：
- 當地田間代理人的直接檢查
- 第三方農藝師審查
- 衛星及遙感資料交叉比對
- 文件真實性驗證

**產出：** 里程碑於鏈上確認，觸發 EscrowVault 資金釋放。

### 第四層 — 財務對帳

**時機：** 收穫後、收益分配前
**內容：** 驗證發起人的回款是否符合約定條款

本層確保財務結算準確且完整：

- 確認發起人的回報存入金額與約定金額一致
- 實際銷售收入與預計回報的對帳
- 平台費用計算及扣除驗證
- ClaimVault 餘額確認後開放領取

**產出：** ClaimVault 已注資，代幣持有者可開始領取。

### 第五層 — 持續監控與 Oracle 整合

**時機：** 批次生命週期全程
**內容：** 超越離散里程碑檢查點的持續風險監控

本層在里程碑檢查點之外提供持續性的監督：

- **天氣與氣候監控** — 追蹤可能影響作物表現的極端天氣事件、乾旱、洪災或病蟲害
- **市場價格監控** — 追蹤與批次作物類型相關的大宗商品價格
- **發起人行為監控** — 異常活動模式、通訊中斷或其他預警信號
- **Oracle 整合** — 在可用的情況下，整合第三方資料預言機以提供自動化數據饋送（天氣資料、大宗商品價格、衛星指數）

**產出：** 風險預警及 Aurora Labs 操作者的早期介入觸發。

## 驗證不等於擔保

必須明確指出，驗證降低風險但不消除風險。Aurora Protocol 的驗證架構提供：

- 對發起人和操作合法性的**合理保證**
- 基於證據的里程碑進度**確認**
- 對新興風險的**持續監控**

但**不提供**以下保證：

- 發起人必然還款的擔保
- 對作物歉收、自然災害或市場崩盤的保險
- 鏈下協議的法律執行力（由獨立法律框架處理）

詳見[風險揭露](../Risks.md)中有關殘餘風險的完整討論，以及[發起人擔保](../Economics/Originator-Security.md)中 $AUR 質押機制提供的額外損失緩解措施。

## 現行狀態與未來規劃

| 層級 | 現行實作 | 規劃提升 |
|---|---|---|
| 第一層 — 身份 | Aurora Labs 手動驗證 | 整合第三方 KYC/AML 服務 |
| 第二層 — 資產 | 手動考察 + 文件審查 | 衛星影像 API 整合 |
| 第三層 — 里程碑 | 手動證據審查 | 自動化證據評分；去中心化驗證網路 |
| 第四層 — 財務 | 手動對帳 | 自動化 USDC 流向追蹤 |
| 第五層 — 監控 | 手動監控 | Oracle 自動化預警；IoT 感測器整合 |

Aurora Protocol 的驗證架構設計目標是隨時間推移逐步實現更高程度的自動化與去中心化，同時在關鍵決策點維持人工監督。
