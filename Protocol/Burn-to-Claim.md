# 銷毀領取（Burn-to-Claim）

銷毀領取機制是 Aurora Protocol 的收益分配系統。該機制透過 **ClaimVault** 合約實作，為代幣持有者提供確定性、防篡改的按比例收益贖回方式。

---

## 機制概覽

批次完成所有三個里程碑後，發起人將約定的回報金額（本金 + 收益）存入 ClaimVault。代幣持有者隨後**銷毀**其 ERC-1155 RWA 代幣，以領取按比例分配的資金。

核心原則簡潔明瞭：**銷毀一枚代幣 = 結算一份按比例的權利**。銷毀操作永久消滅該代幣，確保每份權利僅可行使一次。

---

## 領取流程

```
   發起人                    ClaimVault               代幣持有者
     │                          │                          │
     │  存入回報（USDC）        │                          │
     │─────────────────────────►│                          │
     │                          │                          │
     │                          │   銷毀 RWA 代幣          │
     │                          │◄─────────────────────────│
     │                          │                          │
     │                          │   轉移 USDC              │
     │                          │─────────────────────────►│
     │                          │                          │
```

### 逐步說明

1. **回報存入** — 發起人將完整回報金額（本金 + 收益）以 USDC 轉入 ClaimVault 合約。

2. **發起領取** — 代幣持有者呼叫 ClaimVault 的 `claim()` 函式，指定欲銷毀的代幣數量。

3. **代幣銷毀** — ClaimVault 呼叫 AuroraRWA1155 合約的 `burn()` 函式，從持有者餘額中永久銷毀指定數量的代幣。

4. **USDC 轉移** — ClaimVault 計算並轉移按比例的 USDC 金額至持有者錢包。

---

## 領取計算

每位持有者的領取金額依以下公式計算：

```
領取金額 = (銷毀代幣數 / 總代幣供應量) × 合約總餘額
```

其中：
- `銷毀代幣數` = 持有者在本次交易中銷毀的 RWA 代幣數量
- `總代幣供應量` = 該批次在領取時的 RWA 代幣總供應量
- `合約總餘額` = ClaimVault 中為該批次存入的 USDC 總額

**範例：**
- 代幣總供應量：10,000
- 合約總餘額：105,000 USDC（100,000 本金 + 5,000 收益）
- 持有者銷毀 500 枚代幣
- 領取金額：(500 / 10,000) × 105,000 = **5,250 USDC**

---

## 設計特性

### 防止重複領取

銷毀機制本質上防止重複領取。代幣一經銷毀，即從持有者餘額和總供應量中永久移除，無法重新鑄造或恢復。

### 部分領取

持有者無需一次銷毀所有代幣，可在不同時間執行多次部分領取。每次領取均根據當時的剩餘合約餘額和剩餘代幣供應量重新計算比例份額。

### 原子化執行

每筆領取交易均為原子操作——銷毀和 USDC 轉移在同一筆交易中完成。若任一操作失敗，整筆交易回滾。不存在代幣已銷毀但資金未轉移的中間狀態。

### 無到期限制

領取無截止期限。代幣持有者可在合約獲得資金後的任何時間銷毀並領取。未領取的資金將無限期保留在 ClaimVault 中。

---

## 邊界情況

| 情境 | 行為 |
|---|---|
| 持有者以零代幣嘗試領取 | 交易回滾 |
| 持有者嘗試銷毀超過其餘額的代幣 | 交易回滾 |
| ClaimVault 餘額為零 | 交易回滾（無資金可分配） |
| 發起人存入部分回報 | 持有者領取已存入金額的按比例份額 |
| 所有代幣已銷毀，合約殘留微量餘額 | 由整數取整處理；殘留金額可忽略不計 |

---

## 為何選擇銷毀領取

曾評估其他分配機制，但均因各自缺陷被排除：

| 方案 | 問題 |
|---|---|
| 空投（推送式） | Gas 成本隨持有者數量線性增長；發送方承擔全部 Gas |
| Merkle 領取（快照式） | 需鏈下快照生成；增加信任假設 |
| 按比例串流 | 對離散批次結算而言過度設計 |
| **銷毀領取（拉取式）** | **每位持有者自付 Gas；無需快照；透過代幣銷毀自我執行** |

銷毀領取模式是 Aurora 批次結算模型中最具資金效率、最小化信任需求且 Gas 分攤最公平的分配方式。
