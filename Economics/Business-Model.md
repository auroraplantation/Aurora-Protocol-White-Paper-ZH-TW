# 商業模式

Aurora Protocol 透過單一、透明的費用機制產生收入：**平台費用**。不存在隱藏收費、代幣提取機制或投機性收入依賴。

---

## 平台費用

平台費用為基於百分比的費率，適用於 EscrowVault 每個里程碑釋放的毛資金額。

| 參數 | 數值 |
|---|---|
| 費用類型 | 釋放資金的百分比 |
| 最高費率 | 3% |
| 典型費率 | 1.5% 至 3%（依批次而異） |
| 承擔方 | 發起人（從里程碑撥付中扣除） |
| 收取時點 | 每次里程碑資金釋放時 |
| 收款方 | Aurora Labs 營運錢包 |

### 核心特性

**發起人承擔：** 平台費用從發起人的撥付中扣除，而非從參與者資金中收取。參與者的回報計算基於發起人的毛還款義務，該義務已包含費用因素。

**依批次調整：** 費率於批次建立時設定並編碼於智能合約中。可依批次規模、發起人風險特徵及合作歷史等因素在批次之間調整。所有批次的最高費率上限為 3%。

**里程碑對齊：** 費用僅在價值實際創造時收取——即里程碑確認且資金釋放時。若批次在某里程碑前失敗，未釋放的批次不產生費用。

---

## 收入機制對齊

Aurora Protocol 的商業模式刻意與所有參與方的利益保持一致：

**與參與者對齊：** Aurora Labs 僅在里程碑完成時獲取費用收入。若批次失敗，未釋放的批次不產生費用。這為 Aurora Labs 創造了執行嚴格驗證及篩選高品質發起人的直接經濟動機。

**與發起人對齊：** 費用為批次開始前即確定的已知固定成本。不存在浮動利差、隱藏收費或追溯調整。發起人可在承諾前精確計算其資金成本。

**無代幣依賴：** Aurora Labs 的核心收入模式不依賴任何代幣的增值或交易量。平台永續性根植於真實經濟活動——由農業融資操作中的資金流產生費用。

---

## Aurora Protocol 不收取的費用

| 費用類型 | 狀態 |
|---|---|
| 參與者入場/退出費 | 無 |
| 代幣轉移費 | 無 |
| 領取/銷毀費 | 無 |
| 訂閱或存取費 | 無 |
| 收益績效費 | 無 |
| 發起人登錄費 | 無 |

## 未來收入考量

隨著協議成熟，可能引入額外收入來源。以下項目目前尚未實作，未來將經過治理審議及社群討論：

- **二級市場手續費** — RWA 代幣二級交易的小額交易費（待二級市場啟用後）
- **數據與分析服務** — 面向機構參與者的進階數據饋送或報告
- **驗證即服務** — 將 Aurora 的驗證基礎設施提供予第三方 RWA 協議使用

任何新的費用機制將提前公告，並反映於更新後的協議文件中。
